Definitions

  • Instantiation

    In Computer Science, instantiation is the act of create a real instance of a class object.

    For example, in Java, if there is a class called Shape:
    public class Shape {...}
    You can create an instance of the class by writing the following code:
    Shape rectangle = new Shape();

  • Object

    Definition: An object is an instance of a class. It can also be defined as an abstract data type.

    For example, based on our previous example instantiation, the following code creates/defines an object from the Shape class:

    Shape rectangle = new Shape();

  • Class

    Definition: A class is commonly used in object oriented programming and it is a template that defines methods and variables for a specific object.

    For example, in Java, a class called Shape can be created using the following code:

    public class Shape {
    int num_sides;
    String name;
    ...
    public int get_sides() {
    return num_sides
    }
    ...
    }

  • Namespace

    Definition: In programming, a namespace is a region of code that can declared which creates a scope for names. In other words, it allows for users to define functions that have the same names as other functions.

    For example, in c++ namespaces can be used to accomplish the following:

    namespace other_one {
    int value = 1;
    }
    int main() {
    int value = 2;
    ...
    }

  • Constructor

    Definition: A constructor is a specific method of a class that is used to initialize a newly created object.

    For example, a constructor for the Shape class that we defined in the previous definitions would be:

    public class Shape {
    int num_sides;
    String name;
    public Shape(String name, int num_sides) {
    this.name = name;
    this.num_sides = num_sides;
    }
    public int get_sides() {
    return num_sides
    }
    ...
    }

  • Fixture

    In programming, a fixture is a function that can be used to define a set of steps and data that can be used for testing. It also helps by providing the database with data, database connections, etc.

    An example of an fixture can be shown through the calculator program:

    @pytest.fixture
    def setup_addition_calculation_fixture():
     values = (1, 2)
     addition = Addition(values)
     History.add_calculation(addition)

  • Type Hint

    Type Hint is used in Python to statically check the data type or type of value that is used within your program.

    An example of type hinting can be shown using the calculator program:

    @classmethod
    def create(cls, tuple_list: tuple):
      return cls(tuple_list)

    The example uses type hinting because of the following line of code "def create(cls, tuple_list: tuple):". By writing "tuple_list: tuple", it ensures that the parameter "tuple_list" has a data type of tuple.

  • Type Cast

    A type cast is when the data type of a variable is changed to another data type.

    An example of type casting can be shown using the calculator program:

    @staticmethod
    def convert_args_to_tuple_of_float(tuple_list):
       list_values_float = []
       for item in tuple_list:
         list_values_float.append(float(item))
       return tuple(list_values_float)

    The example uses type casting because in the line of code "list_values_float.append(float(item))" it converts the data type of "item" into a float. Therefore, it is converting every value in tuple_list to a float.

  • Unit Test

    A unit test is the process of testing a unit or logically isolated piece of code or system in a program. The unit can be a function, subroutine, method, property, and several others. It checks to make sure that the part of that system is working efficiently and expectedly.

  • Static

    In programming, static is a modifier that can modify the scope and behavior of methods and properties/variables. If it is added to a method or property then that means that the variable or method belongs to the class rather than a specific instance.

    Examples of static methods and properties can be found in its respective definition.

  • Instance Method

    An instance method is created when removing the static keyword from the method heading. It is a method that belongs to the instances or objects of a class but not the class itself. Therefore, it can refer to instance variables and methods that belong to its object and it can use static variables and methods.

    An example of an instance method can be shown through the calculator program:

    class Addition(Calculation):
      def get_result(self):
        sum_of_values = 0.0
        for value in self.values:
          sum_of_values = Add.add(value, sum_of_values)
        return sum_of_values

  • Instance Property

    A instance property is a property that is defined within a class. Therefore, in each instance of the class the property can be accessed.

    An example of an instance property can be shown through the calculator program:

    class Calculations:
      history = []

    The instance property in the example above is the defined in the line of code "history = []".

  • Static Method

    Static Methods are methods that can be called without having to create an instance or object of a class. Depending on the programming language this can be done by writing "static" in the method definition or in Python "@staticmethod" would have to be written.

    An example of a static method can be shown through the calculator program:

    @staticmethod
    def count_history():
     return len(Calculations.history)

  • Static Property

    A static property is a property that can accessed globally in a program. Static properties can be defined by writing "static" when the property is being declared.

    For example, in Java in order to define a static property or variable you can use the following format:

    static int count = 0;

    By adding static to the properties declaration it allows for the property or variable to become a static property.

  • Encapsulation

    In OOP, encapsulation is commonly used in the form of a class. Overall, it is the concept of bundling data and methods within a single unit. It allows for specific or internal information to be hidden and for the access to that information to be controlled. By using encapsulation users of classes don't know how the classes are implemented or how the information is stored however they can access the logic of the classes through instantiation and class methods.

  • Inheritance

    Inheritance in object oriented programming allows for programmers to derive a class from another class which creates a hierarchy of classes. This allows for classes to share attributes and methods. Since a class can be derived from another class, the class that the class is being derived from is called a superclass or a parent class. While the derived class is called a subclass or child class.

    Polymorphism is the concept which allows for variables, functions, or objects to have multiple different forms.

  • Abstraction

    Abstraction in OOP is a concept that allows for only essential attributes to be shown and it hides the unnecessary information from users. Its assists with handling or reducing the complexity of programs, which helps users to implement more complex logic alongside the abstraction without having to deal with the complexity of the program. So it mainly shows the user how to implement the program but not the complexity of its implementation.

  • Overview of SOLID

    SOLID is an acronym for five object oriented design principles that were made by Robert C. Martin. These principles allow for programmers to adopt a more organized programming style. They also assist in avoiding messy programs, the need for refactoring code, etc. The S stands for Single-Responsibility Principle. The O stands for Open-Closed Principle. The L stands for Liskov Substitution Principle. The I stands for Interface Segregation Principle. The D stands for Dependency Inversion Principle.

    Examples of SOLID and information about the topic can be found here: Click Here

  • Overview of Design Patterns

    Design patterns are programming strategies that can be used when solving a problem that is common in programming. It can be a series of steps or a way to designed or structure a section of code depending on its purpose.

    An example of an design pattern can be shown through the calculator program:

    @classmethod
    def create(cls, tuple_list: tuple):
       return cls(tuple_list)

    The code above is an example of the factory design pattern because it creates an object without exposing the logic to users.
