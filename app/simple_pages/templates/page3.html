Description of Pytest/Examples:

Pytest allows developers to create and run tests on their program in order to ensure that the output that the software is outputting is expected/wanted. By running pytest on their software in between making changes it allows developers to distinguish if the previous changes that were ge ade unknowingly created an error in their program. For example, in my current program, I am creating a web application that involves creating several web pages. I have used pytest to ensure that the content that is being displayed on the pages are expected and whether the proper context processors messages are being displayed properly. In order to use pytest, you would first have to install the package on your computer. Afterwards, you would create a directory/folder called tests to store your test files.

How to Create/Test simple_pages:

In order to create the web pages, I initially created a “base.html” file to create a template that the rest of my web pages will build off of. The file will have the navbar, link to css file, script to link to bootstrap, footer, etc. I created a directory in my app directory called “simple_pages”. This directory contains the templates for the html web pages. I tested these pages using the assert function to make sure that the content that is on the pages are correct.

Outline:

  • docker_flask <- Root directory
  • .github and workflows <- Directory that has workflow files
  • app <- directory that contains files for web pages
    • context_processors <- directory that contains files to define the context processors
      • __init__.py <- file where context processors are defined
  • simple_pages <- directory that contains the html files that will be displayed
    • templates <- directory where html files are
    • __init__.py <- file where page routes are defined and pages are organized according to the templates
  • static <- directory where css files, images, and javascript files are
  • Templates <- directory where base.html file is defined
    • base.html <- html file that contains that basis for the other html files in the project, it includes the navigation bar, link to css file, link to bootstrap, etc.
  • __init__.py <- file creates and configures the flask application and it registers the simple_pages as a blueprint and the context_processors so that they can be used throughout the application
  • run.py <- file which allows Gunicorn to put the app into production
  • tests <- directory where test files are held where pytest can find them
    • __init__.py <- although file is empty, file is there to allow application to run smoothly without error
    • conftest.py <- file that setups the test configuration such as creating the app, making an http client, and creating a task runner
    • context_process_text.py <- file that tests that the context processors are properly displayed on the files
    • simple_pages_test.py <- file that tests that the proper responses are coming from the web pages, to ensure that the routes are correct
  • .coveragerc <- file that is used to assist in configuring application
  • .dockerignore <- file that tells docker which files not to push to docker hub
  • .gitignore <- file that excludes specific files from being pushed to github
  • .pylintrc <- file that is used to assist in configuring application
  • docker-compose.yml <- file that contains the services and configuration that the application needs to develop locally
  • Dockerfile <- file used to build/create your Docker image, the image will contain the dependencies that the Python application needs to build
  • heroku.yml <- file used to define heroku app and allow application to be deployed on heroku
  • pytest.ini <- file that contains configuration for pytest
  • Readme.md <- file that contains project instructions
  • requirements.txt <- file that contains packages that need to be installed for this application
  • setup.py <- file that find and installs the packages that are needed for the application
