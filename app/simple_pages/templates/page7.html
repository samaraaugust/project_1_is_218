Object Oriented Programming

Encapsulation

In OOP, encapsulation is commonly used in the form of a class. Overall, it is the concept of bundling data and methods within a single unit. It allows for specific or internal information to be hidden and for the access to that information to be controlled. By using encapsulation users of classes don't know how the classes are implemented or how the information is stored however they can access the logic of the classes through instantiation and class methods.

An example of encapsulation based on our calculator program would be:

class Calculations:
  history = []
  @staticmethod
  def clear_history():
    Calculations.history.clear()
    return True
...

This is an example of encapsulation because the program creates a class called Calculations and it defines class properties such as history which is an array and it also defines class methods such as clear_history which can be used by users once they create an instance of the class.

Inheritance

Inheritance in object oriented programming allows for programmers to derive a class from another class which creates a hierarchy of classes. This allows for classes to share attributes and methods. Since a class can be derived from another class, the class that the class is being derived from is called a superclass or a parent class. While the derived class is called a subclass or child class.

An example of inheritance can be shown in the calculator program:

class Calculation:
  @classmethod
  def create(cls, tuple_list: tuple):
    return cls(tuple_list)

class Addition(Calculation):
  def get_result(self):
    sum_of_values = 0.0
    for value in self.values:
      sum_of_values = Add.add(value, sum_of_values)
    return sum_of_values

The Calculation class is a parent class and the Addition class is a child class. The Addition class uses the Calculation class as a parameter which causes for the Addition class to because a child of the Calculation class which allows for methods and properties to be shared amongst the classes.

Polymorphism

Polymorphism is the concept which allows for variables, functions, or objects to have multiple different forms.

An example of polymorphism can be shown through the calculator program:

class Calculation:
  @classmethod
  def create(cls, tuple_list: tuple):
    return cls(tuple_list)

class Addition(Calculation):
  def get_result(self):
    sum_of_values = 0.0
    for value in self.values:
      sum_of_values = Add.add(value, sum_of_values)
    return sum_of_values

class Multiplication(Calculation):

This is an example of polymorphism because the class Calculation is the parent class for both the Addition and Multiplication classes. The Multiplication and Addition classes defines the same method get_result(self), however, the methods are altered according to the purpose of the classes despite having the same name.

Abstraction

Abstraction in OOP is a concept that allows for only essential attributes to be shown and it hides the unnecessary information from users. Its assists with handling or reducing the complexity of programs, which helps users to implement more complex logic alongside the abstraction without having to deal with the complexity of the program. So it mainly shows the user how to implement the program but not the complexity of its implementation.
