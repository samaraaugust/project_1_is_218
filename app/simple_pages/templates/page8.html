SOLID

Overview

SOLID is an acronym for five object oriented design principles that were made by Robert C. Martin. These principles allow for programmers to adopt a more organized programming style. They also assist in avoiding messy programs, the need for refactoring code, etc. The S stands for Single-Responsibility Principle. The O stands for Open-Closed Principle. The L stands for Liskov Substitution Principle. The I stands for Interface Segregation Principle. The D stands for Dependency Inversion Principle.

Single-responsibility Principle (SRP) states that "A class should have one and only one reason to change, meaning that a class should have only one job." (1) Open-closed Principle (OCP) encourages programmers to write classes or entities of code in a certain way so that if new functionality is need it the existing code does not need be modified. Therefore, it should be open for extension but closed for changes. Liskov Substitution Principle promotes the concept that if the superclasses of a program were replaced with subclasses, the program wouldn't break or crash because the subclasses behave in the same way that the superclasses does. Interface Segregation Principle promotes the concept of ensuring that every part of a program is necessary therefore, declaring methods that are unnecessary should not occur. In other words, a client should not encounter methods that aren't vital or unnecessary to the program. Dependency Inversion Principle states that "High-level modules, which provide complex logic, should be easily reusable and unaffected by changes in low-level modules, which provide utility features." (2) This can be achieved by using abstraction. The high level modules are parts of a program that is more complex and its more abstract. a low level module is a part of a program that contain specific components and are less complex. The calculator program implements theses principles where for the Interface Segregation Principle the calculator program follows this principle because each method contributes to the program. Also, an example of SRP is in the calculator program is shown in the following:

class Subtraction:
  @staticmethod
  def subtract(value_1, value_2):
    return value_1 - value_2

This example showcases the SRP design principle because it performs only one job which is to subtract two inputs from each other.

Design Patterns

Design patterns are programming strategies that can be used when solving a problem that is common in programming. It can be a series of steps or a way to designed or structure a section of code depending on its purpose.

An example of an design pattern can be shown through the calculator program:

@classmethod
def create(cls, tuple_list: tuple):
   return cls(tuple_list)

The code above is an example of the factory design pattern because it creates an object without exposing the logic to users.
